# Single source of truth for every AI CLI config path that needs to survive
# a container rebuild, read by three scripts (host-side seedHostConfig.sh,
# container-side postCreate.sh and syncConfigOut.sh) so the list is never
# duplicated. One path per line, relative to $HOME.
#
# Whether a given path ends up bind-mounted (live, no copying needed) or
# copied/synced (staged through .agent-sync) is decided at runtime, not by
# anything recorded here: postCreate.sh and syncConfigOut.sh each check
# whether a path is actually mounted (see is_bind_mounted() in both) rather
# than trusting a label here. A label can drift out of sync with
# devcontainer.json's "mounts" array -- a path added here without a
# matching mounts entry, or a mount that fails to attach -- and silently
# stop being synced with no error. Checking the live filesystem instead
# means a missing mount degrades to the slower copy/sync path instead of
# losing data outright. seedHostConfig.sh, which runs on the host before
# any mount exists, decides the same way but against the real host path:
# [ -d ] for a directory, anything else treated as a file.
